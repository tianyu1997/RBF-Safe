# Provenance

RBF-Safe is a new MIT-licensed implementation informed by the algorithms and
tests in RapidBoxForest. No legacy source file is copied unchanged.

The affine-arithmetic DH propagation and paired link-envelope algorithms are
restructured from the authors' MIT-licensed RapidBoxForest implementation into
dimension-safe value-based APIs. LECT retains the deterministic binary
partition concept but replaces legacy storage, numeric identities, journals,
and planner adapters. All other v0.1 implementation is new.

`data/golden/legacy_geometry_v1.json` was generated on 2026-07-21 from a
read-only RapidBoxForest `link_interval_envelope` reference build. It records
point-FK and IFK-AA endpoint bounds for the legacy 2-DOF, IIWA14, and UR5
models. Runtime tests consume only the checked-in fixture and never access the
legacy checkout or its binaries. The UR5 legacy file contained a seventh
unused radius without a tool transform; RBF-Safe intentionally records only
the six represented links.

The v0.3 OMPL adapter is a new implementation against OMPL's public extension
interfaces. RapidBoxForest's experiment-oriented OMPL helpers were consulted
for behavioral context only; their planner wrappers, private-member access,
deterministic subclasses, collision checker, and Python orchestration code are
not copied or migrated.

The v0.4 corridor layer is also a new implementation. Legacy OBB path-cover,
portal overlay, and HiPaC query-bridge files were reviewed only to identify
behavioral concepts and failure modes. RBF-Safe does not copy their Eigen
types, GJK/zonotope implementation, planner-private access, adaptive sweep
configuration, diagnostics, or promotion logic. Its OBBs use standard
containers, conservative AABB-enclosure proofs, bounded recursive covering,
and new subject-bound certificate identities.

The v0.5 Safe IK solver and MoveIt 2 package are new implementations. The
legacy Safe IK and integration notes were used only to establish the intended
high-level contract. RBF-Safe uses its own pose type, Atlas route evidence,
bounded projected numerical method, current MoveIt 2 adapter split, and ROS 2
Jazzy `KinematicsBase` API; no legacy ROS, planner, or IK source was copied.

The v0.6 dynamic update and version-store implementations are new. Legacy
dynamic-update notes were used only to identify product requirements. The new
code uses schema-2 subject binding, canonical obstacle-ID deltas, stored link
envelopes, bounded local LECT repair, and atomic immutable version directories.
No legacy invalidation journal or cache layout is reused.

The v0.7 generalized region database, half-space Portal solver, OBB Atlas
builder, and correlation-preserving Taylor backend are new implementations.
Legacy zonotope, portal-overlay, and planner-growth code was reviewed only for
product vocabulary and test scenarios. RBF-Safe uses new standard-container
types, deterministic subject identities, explicit budgets, conservative
Taylor remainders, and an independent validated storage schema. No legacy
Eigen/GJK representation, planner adapter, cache page, or source file was
copied.

The v0.8 planning and optimization consumers are new implementations against
the public RBF-Safe and upstream framework APIs. The certified sampler,
roadmap construction, OMPL orchestration, MoveIt constraint sampler, linear
constraint compiler, projection routines, and solver-named front ends were
written for this repository. No RapidBoxForest planner wrapper, sampling
adapter, optimizer integration, or MoveIt source was copied.

The v0.9 runtime shield is a new implementation against the public RBF-Safe
Atlas, trajectory-auditor, and Safe IK APIs. The original product sketch was
used for the action-gate vocabulary and VLA integration goal. The bounded
projection, route composition, decision identity, batch policy, telemetry,
and observation monitor were written for this repository; no legacy runtime,
policy, planner, or robot-control source was copied.

The v1.0 release benchmark, compatibility gate, and named release fixtures are
new repository-local release assets. Robot parameters were transcribed into
the public RBF-Safe JSON schema for deterministic API regression coverage; the
scene geometry is deliberately synthetic and far-field. These files were not
copied from RapidBoxForest experiments and do not claim calibration or
deployment fidelity.

The v2.0 learning-policy gate and feedback database are new implementations
against the public v0.9 runtime shield. The product sketch supplied metadata,
policy-learning, and safety-memory goals only. Proposal identity, deterministic
threshold/selection logic, feedback semantics, validation, schema, bindings,
tools, examples, and tests were written for this repository; no RapidBoxForest
policy, learning, runtime-control, or persistence source was copied.

`data/atlas_schema1` is a fixed Atlas generated by RBF-Safe v0.5 commit
`cc26a87` using its public quickstart. It proves v0.6 schema-1 loading and
round-trip preservation without depending on a previous binary at test time.
